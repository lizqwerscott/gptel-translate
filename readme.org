* Overview
~gptel-translate~ is an Emacs package that translates a buffer (or a selected region) paragraph-by-paragraph using [[https://github.com/karthink/gptel][gptel]]. It creates a dedicated result buffer where the original text and the LLM-produced translation are displayed side by side, with syntax highlighting and navigation bindings.

* Installation
** With ~elpaca~
#+begin_src emacs-lisp
  (elpaca (gptel-translate :fetcher github :repo "lizqwerscott/gptel-translate" :files (:defaults "prompts")))
#+end_src

** Manual
Clone the repository and add it to ~load-path~:

#+begin_src sh
git clone https://github.com/lizqwerscott/gptel-translate ~/.emacs.d/site-lisp/gptel-translate
#+end_src

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/gptel-translate")
(require 'gptel-translate)
#+end_src

* Usage
** Commands

| Command                         | Description                                                     |
|---------------------------------+-----------------------------------------------------------------|
| ~M-x gptel-translate-buffer~    | Translate whole buffer, or active region if one is selected     |
| ~M-x gptel-translate-at-point~  | Translate current paragraph (non-Org) or current Org subtree    |

- ~gptel-translate-buffer~: With an active region, only that region is translated.
  In Org mode, content is collected by subtree (headings + body paragraphs).
  In other modes, paragraph-based collection is used.

- ~gptel-translate-at-point~: Translates only the content at point.
  In Org mode, this translates the current subtree (heading + all children).
  Repeated calls on the same scope reuse the same result buffer.
  In non-Org modes, translates the current paragraph.

** Key bindings in the result buffer

| Key               | Command                                           |
|-------------------+---------------------------------------------------|
| ~RET~             | Jump to original in source buffer                 |
| ~TAB~ / ~n~       | Next paragraph                                    |
| ~S-TAB~ / ~p~     | Previous paragraph                                |
| ~g~               | Re-translate all content (~gptel-translate-retranslate~) |
| ~C-g~             | Abort translation (~gptel-translate-abort~)             |

When moving between paragraphs, if ~gptel-translate-followp~ is non-nil (the
default), point in the source buffer follows along automatically.

~gptel-translate-retranslate~ re-collects paragraphs from the source buffer
(picking up any edits you may have made) and re-sends all paragraphs to the LLM,
clearing existing translations.

~gptel-translate-abort~ stops any in-progress translation.
